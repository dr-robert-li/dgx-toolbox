Key facts about autoresearch:
- Uses `uv` package manager, Python 3.10, PyTorch + CUDA
- prepare.py: downloads data, trains BPE tokenizer (immutable, ~2 min on H100)
- train.py: GPT model + Muon/AdamW optimizer — agent edits this file
- program.md: agent instructions — user points Claude/agent here
- Agent loop: read program.md -> modify train.py -> run 5min training -> eval val_bpb -> commit or reset -> repeat
- Default params tuned for H100 (16,896 cores). DGX Spark has 128 Blackwell cores.
- prepare.py has constants: MAX_SEQ_LEN, DEVICE_BATCH_SIZE, EVAL_TOKENS, vocab_size
- train.py has: DEPTH, TOTAL_BATCH_SIZE, WINDOW_PATTERN, learning rate

DGX Spark hardware:
- NVIDIA Blackwell GPU, 128 CUDA cores (not a data center GPU)
- aarch64 ARM64 architecture
- ~1TB NVMe storage
- PyTorch 2.9.1 + CUDA 12.8

**spark-config.sh** — DGX Spark tuning overrides sourced by both launchers:
```
#!/usr/bin/env bash
# DGX Spark tuning for 128 Blackwell CUDA cores
# H100 has 16,896 cores — Spark has ~0.75% of that compute.
# These overrides are applied via sed to train.py after clone/pull.

# --- Model Architecture ---
SPARK_DEPTH=4              # Reduce from default (likely 12+) — fewer transformer layers
SPARK_TOTAL_BATCH_SIZE=4   # Reduce from default — fits in limited GPU memory
SPARK_DEVICE_BATCH_SIZE=1  # Micro-batch size per step

# --- Sequence Length ---
SPARK_MAX_SEQ_LEN=256      # Reduce from default (likely 1024+) — less memory per sample

# --- Training Duration ---
SPARK_TRAIN_MINUTES=10     # Increase from 5min — slower GPU needs more time per experiment

# --- Learning Rate ---
SPARK_LR_SCALE=0.5         # Scale factor applied to default learning rate (slower convergence safer on small GPU)

# --- Gradient Accumulation ---
SPARK_GRAD_ACCUM=8         # Accumulate gradients to simulate larger effective batch

# --- Eval ---
SPARK_EVAL_TOKENS=100000   # Reduce eval token count for faster eval cycles
```

Include a function `apply_spark_config()` that:
1. Takes a train.py path as argument
2. Uses sed to patch the known parameter names (DEPTH, TOTAL_BATCH_SIZE, etc.)
3. Also patches prepare.py constants (MAX_SEQ_LEN, DEVICE_BATCH_SIZE, EVAL_TOKENS) if present
4. Prints what was changed for transparency
5. Is idempotent — safe to run multiple times

Also include `apply_spark_timing()` that patches the 5-minute wall-clock timer in train.py to SPARK_TRAIN_MINUTES.

**launch-autoresearch.sh** — Interactive launcher:
```
#!/usr/bin/env bash
set -euo pipefail
```

Structure:
1. Source lib.sh (use `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"` then `source "$SCRIPT_DIR/../lib.sh"`)
2. Set AUTORESEARCH_DIR="${HOME}/autoresearch" (working clone location, NOT inside dgx-toolbox)
3. Clone or pull latest master:
   - If $AUTORESEARCH_DIR/.git exists: `cd $AUTORESEARCH_DIR && git pull origin master`
   - Else: `git clone https://github.com/karpathy/autoresearch.git $AUTORESEARCH_DIR`
4. Check `uv` is installed, if not: `curl -LsSf https://astral.sh/uv/install.sh | sh && export PATH="$HOME/.local/bin:$PATH"`
5. `cd $AUTORESEARCH_DIR && uv sync`
6. **Data source selection** — interactive menu using `select`:
   ```
   echo "Select training data source:"
   select src in "Default (autoresearch built-in)" "Local directory" "Hugging Face dataset" "GitHub repo" "Kaggle dataset"; do
   ```
   For each option:
   - **Default**: Run `uv run prepare.py` with no changes
   - **Local directory**: Prompt for path, validate it exists, symlink or copy .txt/.parquet files into data/ dir, then run prepare.py
   - **Hugging Face**: Prompt for dataset name (e.g., "karpathy/climbmix-400b-shuffle"), set `AUTORESEARCH_HF_DATASET` env var, run `uv run prepare.py` (or if prepare.py doesn't support custom datasets, download via `huggingface-cli download <name> --local-dir data/` then run prepare.py)
   - **GitHub**: Prompt for repo URL, clone into a temp dir, copy data files into data/, run prepare.py
   - **Kaggle**: Prompt for dataset identifier (e.g., "user/dataset-name"), download via `kaggle datasets download -d <id> -p data/ --unzip`, run prepare.py. Print warning if kaggle CLI not installed.

   After data prep, always validate that tokenizer output exists before continuing.

7. **Apply DGX Spark tuning**: Source spark-config.sh, call `apply_spark_config "$AUTORESEARCH_DIR/train.py"` and `apply_spark_timing "$AUTORESEARCH_DIR/train.py"`
8. Print banner with: autoresearch dir, data source chosen, GPU tuning applied, estimated experiment duration
9. Print instruction: "Point your AI agent at $AUTORESEARCH_DIR/program.md to begin autonomous research"
10. Optionally ask if user wants to run a single test experiment: `uv run train.py` to validate setup

**launch-autoresearch-sync.sh** — Headless sync-mode launcher:
- No interactive prompts — all config via environment variables:
  - `AUTORESEARCH_DATA_SOURCE` — one of: default, local, huggingface, github, kaggle
  - `AUTORESEARCH_DATA_PATH` — path/name/URL for the selected source
  - `AUTORESEARCH_SKIP_TUNE` — set to "1" to skip DGX Spark parameter tuning
  - `AUTORESEARCH_RUN_TEST` — set to "1" to run one test experiment after setup
- Same logic as interactive launcher but reads env vars instead of prompting
- Prints status messages suitable for log capture
- Does NOT run agent loop — just prepares the environment

Both scripts must be chmod +x.
