**Section 0 — Constants and helpers:**
- SCRIPT_DIR, PROJECT_DIR derivation (same pattern as eval-checkpoint.sh)
- Source lib.sh for shared functions
- DEMO_CYCLES="${DEMO_CYCLES:-3}" — configurable cycle count, default 3
- HARNESS_URL="${HARNESS_URL:-http://localhost:5000}"
- Color/formatting helpers for summary output (use printf, not echo -e)

**Section 1 — Prerequisites check:**
- Verify autoresearch repo exists at ~/autoresearch (if not, clone it — same logic as launch-autoresearch.sh line ~60)
- Check if harness is reachable (curl -sf ${HARNESS_URL}/probe with timeout 5) — warn if not running but continue (screening and eval will skip gracefully)
- Check if vLLM or Ollama is running for inference (informational only)

**Section 2 — Data source selection:**
- Reuse the data source menu from launch-autoresearch.sh. The cleanest approach: call launch-autoresearch.sh with a DEMO_MODE=1 env var that makes it set up data and exit before starting training. However, since launch-autoresearch.sh doesn't support this mode, instead replicate the 6-option select menu inline (copy the menu logic from launch-autoresearch.sh). Present the same options: (1) built-in default, (2) local directory from ~/data/, (3) HuggingFace dataset, (4) GitHub repo, (5) Kaggle, (6) local datasets auto-discovered.
- After selection, record the chosen data source name for the summary.

**Section 3 — Optional data screening:**
- Ask user: "Screen training data through safety harness? (y/N)"
- If yes AND harness is reachable AND a data file path is available, call scripts/screen-data.sh on the data file. Record screened/removed counts for summary.
- If harness not reachable, print warning and skip screening.

**Section 4 — Apply DGX Spark tuning:**
- Source spark-config.sh to apply GPU tuning parameters (same as launch-autoresearch.sh does)
- Set AUTORESEARCH_MAX_CYCLES=$DEMO_CYCLES to limit training to 3 cycles
- Note: autoresearch uses a cycle-based loop. To limit cycles, the script should set the max_cycles parameter. If autoresearch doesn't natively support max_cycles, use a background monitor that counts "Cycle N complete" lines in output and sends SIGINT after $DEMO_CYCLES cycles complete.

**Section 5 — Run autoresearch training:**
- cd ~/autoresearch
- Run `uv run train.py` with appropriate args, capturing output to a log file (tee to both terminal and ${PROJECT_DIR}/demo-training.log)
- The cycle-limiting mechanism: run in background, monitor log for cycle completion pattern, send SIGTERM after DEMO_CYCLES cycles. Use trap to clean up background process on script exit.
- After training completes (or is stopped), find the latest checkpoint directory under ~/autoresearch/experiments/ or wherever autoresearch writes checkpoints.

**Section 6 — Post-training safety eval:**
- Locate the latest checkpoint: find the most recent directory under ~/autoresearch that contains config.json (HF format)
- Call: scripts/eval-checkpoint.sh "$CHECKPOINT_DIR"
- Parse the resulting safety-eval.json for pass/fail and F1 score
- Record results for summary

**Section 7 — Print final summary:**
Print a clear summary block:
```
========================================
  AUTORESEARCH DEMO COMPLETE
========================================
  Dataset:        <data source name>
  Training:       <N> cycles completed
  Screening:      <screened/skipped>
  Safety eval:    <PASS/FAIL> (F1: 0.XX)
  Registered as:  autoresearch/<experiment-name>

  Query the model:
  curl -s -X POST http://localhost:5000/v1/chat/completions \
    -H "Authorization: Bearer sk-devteam-test" \
    -H "Content-Type: application/json" \
    -d '{"model": "autoresearch/<experiment-name>", "messages": [{"role": "user", "content": "Hello"}]}'
========================================
```

If safety eval failed, still print the summary but note the model was NOT registered and omit the curl command. Instead print: "Checkpoint saved at: <path>. Re-run eval with: scripts/eval-checkpoint.sh <path>"

**Error handling:**
- trap EXIT for cleanup (kill background training process if still running)
- Each section checks prerequisites before proceeding
- Script exits 0 on success (even if eval fails — non-destructive per TRSF-02)
- Script exits 1 only on hard errors (clone failure, no checkpoint found)

Make the script executable (chmod +x).

Content structure:

3. **Pipeline stages** — Walk through each stage with the command, expected output, and troubleshooting. Use the same formatting style as the rest of the README (tables, code blocks, clear headers).

**Stage 1: Data Source Selection**
- The demo presents the same 6-option menu as `autoresearch`
- Expected: you see "Select data source:" with numbered options
- Note: for a quick test, option 1 (built-in default) works without any setup

**Stage 2: Training Data Screening (Optional)**
- Requires safety harness running on :5000
- Command shown: `scripts/screen-data.sh <your-data-file>`
- Expected: prints count of clean vs flagged records
- If harness not running: screening is skipped with a warning

**Stage 3: Autoresearch Training**
- Runs for DEMO_CYCLES cycles (default 3, ~8 min each)
- Expected: autoresearch output showing cycle number, loss, eval metrics
- GPU memory: DGX Spark tuning applied automatically via spark-config.sh

**Stage 4: Safety Eval**
- Automatically runs after training completes
- Expected: "PASS" or "FAIL" with F1/precision/recall scores
- Passing checkpoints are auto-registered in LiteLLM config

**Stage 5: Query the Model**
- If eval passed, the summary prints a copy-pasteable curl command
- Show the curl command format:
```bash
curl -s -X POST http://localhost:5000/v1/chat/completions \
  -H "Authorization: Bearer sk-devteam-test" \
  -H "Content-Type: application/json" \
  -d '{"model": "autoresearch/<experiment-name>", "messages": [{"role": "user", "content": "Hello"}]}'
```

4. **Manual pipeline** — For users who want to run stages individually:
```bash
# 1. Launch autoresearch interactively
autoresearch

# 2. Screen data (optional, requires harness on :5000)
scripts/screen-data.sh ~/data/my-dataset/train.jsonl

# 3. After training, evaluate the checkpoint
scripts/eval-checkpoint.sh ~/autoresearch/experiments/<experiment>/checkpoint

# 4. If eval fails, check why
cat ~/autoresearch/experiments/<experiment>/checkpoint/safety-eval.json

# 5. Deregister a model when no longer needed
scripts/autoresearch-deregister.sh autoresearch/<experiment-name>
```

5. **Troubleshooting table:**

| Problem | Solution |
|---------|----------|
| "Harness not reachable" | Start harness: `harness` (port 5000) |
| Training OOM | Reduce batch size in spark-config.sh or set AUTORESEARCH_SKIP_TUNE=1 |
| Safety eval FAIL | Checkpoint is preserved — review safety-eval.json, adjust constitution or thresholds |
| "No checkpoint found" | Check ~/autoresearch/experiments/ for the latest experiment directory |
| Model not queryable after registration | Run `docker restart litellm` to reload config |

6. **No GPU available** — Brief note: "Without a GPU, you can skip training and test the eval/registration pipeline with any existing HF-format checkpoint: `scripts/eval-checkpoint.sh /path/to/checkpoint`"
