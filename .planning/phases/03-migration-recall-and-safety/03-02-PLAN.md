**lib/notify.sh** — Create new file (sourced lib, no set -e):

1. Shebang + header comment. No `set -e`.
2. `notify_user()` function taking 2 args: summary, body.
   - Get uid via `id -u`.
   - Try to find DBUS address:
     a. `gnome_pid=$(pgrep -u "$uid" gnome-session 2>/dev/null | head -1)`.
     b. If gnome_pid found: `dbus_addr=$(grep -z DBUS_SESSION_BUS_ADDRESS "/proc/${gnome_pid}/environ" 2>/dev/null | tr -d '\0' | sed 's/DBUS_SESSION_BUS_ADDRESS=//')`.
     c. Fallback: `dbus_addr="unix:path=/run/user/${uid}/bus"` (systemd user bus socket, always present on Ubuntu 22+).
   - Attempt notify-send: `DISPLAY=":0" XDG_RUNTIME_DIR="/run/user/${uid}" DBUS_SESSION_BUS_ADDRESS="$dbus_addr" notify-send --app-name="modelstore" "$summary" "$body" 2>/dev/null`.
   - If notify-send succeeds (exit 0), return 0.
   - Fallback: append timestamped message to `${HOME}/.modelstore/alerts.log`: `echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] $summary -- $body" >> "${HOME}/.modelstore/alerts.log"`.

8. Pre-recall guard for auto trigger: if `TRIGGER == "auto"`, check `fuser -s "$MODEL_PATH" 2>/dev/null`. If files are in use, `ms_log "Model in use, skipping auto-recall: $MODEL_PATH"` and exit 0.

    - Elif `"$MODEL_PATH" == "${HOT_OLLAMA_PATH}"*`: extract model_name from path, call `ollama_recall_model "$model_name" "$HOT_OLLAMA_PATH"`.
    - Else: `ms_die "Unknown model type: $MODEL_PATH"`.
13. Update state phase: `_write_op_state "recall" "$MODEL_PATH" "cleanup" "$TRIGGER"`.
14. Reset usage timestamp: update usage.json for this model to current timestamp using the same flock+jq pattern from watcher.sh:

    ```bash
    local ts; ts=$(date -u +%Y-%m-%dT%H:%M:%SZ)
    (flock -x 9; jq --arg p "$MODEL_PATH" --arg t "$ts" '.[$p] = $t' "$USAGE_FILE" > "${USAGE_FILE}.tmp" && mv "${USAGE_FILE}.tmp" "$USAGE_FILE") 9>"${USAGE_FILE}.lock"
    ```
15. Compute duration: `end_epoch=$(date +%s); duration=$((end_epoch - start_epoch))`.
16. Get model size for audit: `size=$(du -sb "$MODEL_PATH" 2>/dev/null | cut -f1 || echo 0)`.
17. Audit: `audit_log "recall" "$MODEL_PATH" "${size:-0}" "$COLD_PATH" "$MODEL_PATH" "$duration" "$TRIGGER"`.
18. Clear state: `_clear_op_state`.
19. Log: `ms_log "Recall complete: $MODEL_PATH (${duration}s)"`.

**cron/disk_check_cron.sh** — Create new executable file (`chmod +x`):

1. Shebang + `set -euo pipefail`. Header: "Disk usage threshold check — fires notification when either drive exceeds 98%".
2. SCRIPT_DIR resolution. Source: common.sh, config.sh, notify.sh, audit.sh.
3. `load_config` to get HOT_HF_PATH, COLD_PATH.
4. `check_disk_threshold()` function taking 1 arg (path):
   - Get usage percentage: `pct=$(df --output=pcent "$path" | tail -1 | tr -d ' %')`.
   - Compute drive hash for marker file: `drive_hash=$(echo "$path" | md5sum | cut -d' ' -f1)`.
   - Marker file path: `marker="${HOME}/.modelstore/disk_alert_sent_${drive_hash}"`.
   - If `pct >= 98`:
     - If marker does NOT exist (first time crossing threshold):
       - Get avail and total: `avail=$(df -BG --output=avail "$path" | tail -1 | tr -d ' G')`, `total=$(df -BG --output=size "$path" | tail -1 | tr -d ' G')`.
       - Determine drive label: if path matches HOT_HF_PATH or HOT_OLLAMA_PATH parent, label="Hot storage"; if matches COLD_PATH, label="Cold storage".
       - Call `notify_user "modelstore: disk warning" "${label} at ${pct}% (${avail}GB free / ${total}GB). Run: modelstore migrate"`.
       - Call `audit_log "disk_warning" "$path" 0 "$path" "" 0 "cron"`.
       - Create marker: `touch "$marker"`.
     - If marker exists: skip (suppressed).
   - If `pct < 98`:
     - Remove marker if exists: `rm -f "$marker"` (usage dropped below threshold, re-arm).
5. Main body:
   - `check_disk_threshold "$(dirname "$HOT_HF_PATH")"` (hot drive = parent of HF cache dir).
   - `check_disk_threshold "$COLD_PATH"`.

**watcher.sh** — Extend the existing `watch_inotify()` function. Do NOT rewrite the file. Add cold symlink detection INSIDE the existing while loop, after the `ms_track_usage` call:

1. After `[[ -n "$model_path" ]] && ms_track_usage "$model_path"`, add:
   ```bash
   # Auto-recall: if accessed path is a symlink pointing to cold storage, trigger recall
   if [[ -L "$model_path" ]]; then
     local link_target
     link_target=$(readlink -f "$model_path" 2>/dev/null || true)
     if [[ -n "$link_target" && "$link_target" == "${COLD_PATH}"/* ]]; then
       ms_log "Cold symlink access detected: $model_path -- triggering recall"
       "${SCRIPT_DIR}/../cmd/recall.sh" "$model_path" --trigger=auto 2>/dev/null || \
         ms_log "Auto-recall failed for $model_path"
     fi
   fi
   ```
2. Ensure `COLD_PATH` is available in `watch_inotify()`: `load_config` is already called at the top of the function, which sets COLD_PATH.
