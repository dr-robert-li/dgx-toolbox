  <action>
    1. Create `harness/ratelimit/sliding_window.py` with:
       - `class RateLimitExceeded(Exception)` with `detail` attribute (str)
       - `class SlidingWindowLimiter` with:
         a. `__init__(self)`: `self._rpm_log: dict[str, deque] = defaultdict(deque)`, `self._tpm_log: dict[str, deque[tuple[float, int]]] = defaultdict(deque)`, `self._lock = asyncio.Lock()`
         b. `async def check_rpm(self, tenant_id: str, rpm_limit: int) -> None`:
            - Acquire lock, get current time via `time.monotonic()`, window = 60.0
            - Pop entries older than `now - window` from the left of `self._rpm_log[tenant_id]`
            - If `len(deque) >= rpm_limit`: raise `RateLimitExceeded("RPM limit exceeded")`
            - Append `now` to deque
         c. `async def record_tpm(self, tenant_id: str, tokens: int) -> None`:
            - Acquire lock, append `(time.monotonic(), tokens)` to `self._tpm_log[tenant_id]`
         d. `async def check_tpm(self, tenant_id: str, tpm_limit: int) -> None`:
            - Acquire lock, prune entries older than 60s, sum tokens, raise `RateLimitExceeded("TPM limit exceeded")` if sum > tpm_limit
       - RPM is checked PRE-request (gate). TPM is recorded POST-response and checked PRE-next-request. This means TPM has one-request lag by design (tokens are unknown until LiteLLM responds).

    2. Wire rate limiter into `harness/main.py`:
       - Create `SlidingWindowLimiter()` instance in lifespan, store as `app.state.rate_limiter`

    3. Create `harness/tests/test_ratelimit.py` with tests matching behavior block.
       - Use `monkeypatch` or a custom clock to control `time.monotonic()` for window sliding tests.
       - Each test creates its own `SlidingWindowLimiter()` instance (no shared state between tests).

    4. Create `harness/ratelimit/__init__.py` exporting `SlidingWindowLimiter` and `RateLimitExceeded`.
