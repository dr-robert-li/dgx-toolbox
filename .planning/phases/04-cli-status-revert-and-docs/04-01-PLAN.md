    - _init_revert_state "$total": writes op_state.json with {op:"revert", phase:"recall_hf", started_at:..., trigger:"manual", completed_models:[], total_models:$total}
    - _append_completed "$model": jq --arg m "$model" '.completed_models += [$m]' to op_state.json atomically (write tmp, mv)

    Startup checks (in order):
    1. Check cold drive mounted: check_cold_mounted "$COLD_PATH"
    2. Handle existing op_state.json:
       - If .op != "revert": check age. If < 4 hours, abort: "Another operation in progress (${op}). Wait or check state." If >= 4 hours, clear and proceed.
       - If .op == "revert": resume mode (read completed_models for skip list)

    Collect migrated models:
    - HF: find "$HOT_HF_PATH" -maxdepth 1 -name "models--*" -type l 2>/dev/null | sort
    - Ollama: find cold storage ollama dir for migrated models (check symlinks in HOT_OLLAMA_PATH)

    Preview (unless --force or not TTY):
    - if [[ "$FORCE" != "true" ]] && [[ -t 0 ]]; then print preview table (model name, size, ecosystem), total size, prompt "Proceed? [y/N]", exit 0 if answer != y/Y
    - if [[ "$FORCE" != "true" ]] && [[ ! -t 0 ]]; then ms_die "No TTY for confirmation. Use --force for headless execution."

    Initialize revert state (if not resuming):
    - _init_revert_state "$total_count"

    Recall loop (HF models):
    - For each symlink: check if in completed_models (jq -e --arg m "$model_path" '.completed_models | index($m) != null'). Skip if yes.
    - Call hf_recall_model "$model_path" "$(dirname "$model_path")"
    - _append_completed "$model_path"
    - audit_log "revert" ...

    Recall loop (Ollama models):
    - Similar pattern with ollama_recall_model

    Cleanup phases (each with _write_op_state update):
    - Phase cleanup_cron: crontab -l 2>/dev/null | grep -v "modelstore" | crontab - (only if grep -q finds entries)
    - Phase cleanup_watcher: if [[ -f "$PIDFILE" ]]; then kill "$(cat "$PIDFILE")" 2>/dev/null || true; rm -f "$PIDFILE"; fi
    - Phase cleanup_cold_dir: rm -rf "${COLD_PATH}/hf" "${COLD_PATH}/ollama" 2>/dev/null || true
    - DO NOT remove ~/.modelstore/config.json

    Final:
    - _clear_op_state
    - ms_log "Revert complete"

The adapter rsync calls should respect this (or the adapters already handle it). Document that progress bar integration is handled in Plan 02 Task 1 if adapters need modification.
