Create `modelstore/lib/hf_adapter.sh` with these functions (no `set -e`, sourced library file):

1. **Source common.sh and config.sh** at the top using BASH_SOURCE pattern:
   ```
   _MS_LIB="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   source "${_MS_LIB}/common.sh"
   source "${_MS_LIB}/config.sh"
   ```

2. **hf_list_models()** — prints TSV: `model_path\tsize_bytes` per line.
   - Primary: `python3 -c "from huggingface_hub import scan_cache_dir; ..."` using `repo.repo_path` (absolute path) and `repo.size_on_disk`.
   - Fallback: `for model_dir in "${HOT_HF_PATH}"/models--*/; do du -sb ...`.
   - Must call `load_config` first for HOT_HF_PATH.

3. **hf_get_model_size(model_id)** — prints size in bytes to stdout. `model_id` is the absolute path to `models--*/` dir. Uses `du -sb "$model_id" | cut -f1`.

4. **hf_get_model_path(model_id)** — prints the absolute path to stdout. Since model_id IS the path, just echo it (exists for interface consistency).

5. **hf_migrate_model(model_id, cold_base)** — moves HF model dir to cold storage + creates atomic symlink:
   - Guard 1: `check_cold_mounted "$cold_base"` (calls ms_die if unmounted)
   - Guard 2: `local size; size=$(hf_get_model_size "$model_id"); check_space "$cold_base" "$size"` — return 1 if insufficient
   - Skip if already symlink: `[[ -L "$model_id" ]] && ms_log "Already migrated: $model_id" && return 0`
   - Compute cold target: `local dirname; dirname=$(basename "$model_id"); local cold_target="${cold_base}/hf/${dirname}"`
   - Create cold dir: `mkdir -p "${cold_base}/hf"`
   - rsync: `rsync -a --remove-source-files "$model_id/" "$cold_target/"` (trailing slashes important)
   - Clean empty source dir: `find "$model_id" -type d -empty -delete 2>/dev/null; rmdir "$model_id" 2>/dev/null || true`
   - Atomic symlink: `ln -s "$cold_target" "${model_id}.new" && mv -T "${model_id}.new" "$model_id"`
   - Log: `ms_log "Migrated HF model: $model_id -> $cold_target"`

6. **hf_recall_model(model_id, hot_base)** — moves model back from cold, replaces symlink:
   - Guard: `[[ ! -L "$model_id" ]] && ms_log "Not a symlink, skip recall: $model_id" && return 0`
   - Read target: `local cold_target; cold_target=$(readlink -f "$model_id")`
   - Check cold mounted: `check_cold_mounted "$(dirname "$(dirname "$cold_target")")"` (cold_base is parent of hf/)
   - Check space on hot: `local size; size=$(du -sb "$cold_target" | cut -f1); check_space "$(dirname "$model_id")" "$size"` — return 1 if insufficient
   - Remove symlink: `rm "$model_id"`
   - rsync back: `rsync -a --remove-source-files "$cold_target/" "$model_id/"`
   - Clean cold dir: `find "$cold_target" -type d -empty -delete 2>/dev/null; rmdir "$cold_target" 2>/dev/null || true`
   - Log: `ms_log "Recalled HF model: $cold_target -> $model_id"`

Create `modelstore/lib/ollama_adapter.sh` with these functions (no `set -e`, sourced library file):

1. **Source common.sh and config.sh** at the top using same BASH_SOURCE pattern.

2. **ollama_check_server()** — returns 0 if Ollama server is running, 1 if not:
   - `systemctl is-active --quiet ollama 2>/dev/null && return 0`
   - `curl -sf http://localhost:11434/api/tags >/dev/null 2>&1 && return 0`
   - `return 1`

3. **ollama_list_models()** — prints TSV: `model_name\tsize_bytes` per line:
   - Primary: `curl -sf http://localhost:11434/api/tags | jq -r '.models[] | [.name, (.size|tostring)] | @tsv'`
   - Fallback: `ms_log "WARNING: Cannot reach Ollama API"; return 1`

4. **ollama_get_model_size(model_name)** — prints bytes to stdout:
   - `curl -sf http://localhost:11434/api/tags | jq -r --arg name "$model_name" '.models[] | select(.name == $name) | .size'`

5. **ollama_get_model_path(model_name)** — prints model name (Ollama is API-only, no filesystem paths):
   - `echo "$model_name"` (path concept is the model tag for API-driven operations)

6. **ollama_migrate_model(model_name, cold_base)** — stub with correct guards:
   - Guard 1: `if ollama_check_server; then ms_die "Ollama server is active. Stop it first: systemctl stop ollama"; fi`
   - Guard 2: `check_cold_mounted "$cold_base"`
   - Guard 3: `local size; size=$(ollama_get_model_size "$model_name"); check_space "$cold_base" "$size"` — return 1 if insufficient
   - Stub body: `ms_log "Ollama migration deferred to Phase 3 (ollama cp + ollama rm)"; return 0`

7. **ollama_recall_model(model_name, hot_base)** — stub with correct guards:
   - Guard: `if ollama_check_server; then ms_die "Ollama server is active. Stop it first: systemctl stop ollama"; fi`
   - Stub body: `ms_log "Ollama recall deferred to Phase 3"; return 0`

CRITICAL anti-patterns to avoid:

- Never access `/usr/share/ollama` or `~/.ollama` filesystem paths directly
- No `set -e` in either lib file (sourced, not executed)
- No side effects on source (no echo/exit outside functions)
