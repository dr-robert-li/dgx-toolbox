  <action>
    Create harness/hitl/ui.py with a standalone Gradio app that connects to the harness API.

    1. Imports: gradio as gr, httpx, json, difflib.

    2. Define `def build_ui(api_url: str, api_key: str) -> gr.Blocks:` that:

    a) Creates sync httpx.Client with base_url=api_url, headers={"Authorization": f"Bearer {api_key}"}, timeout=30.0.

    b) Builds gr.Blocks(title="HITL Review Dashboard") layout:

    **Left panel (scale=1):**
    - gr.Dropdown for rail filter: choices=["all", "injection", "pii", "toxicity", "content_filter", "jailbreak"], value="all", label="Rail Type"
    - gr.Dropdown for tenant filter: choices=["all"], value="all", label="Tenant" (populated dynamically)
    - gr.Dropdown for time range: choices=["1h", "24h", "7d", "30d"], value="24h", label="Time Range"
    - gr.Checkbox for hide_reviewed, label="Hide Reviewed", value=False
    - gr.Button("Refresh Queue", variant="secondary")
    - gr.Dataframe for queue table with headers=["Request ID", "Timestamp", "Tenant", "Rail", "Priority", "Action", "Status", "Prompt"], interactive=False, wrap=True

    **Right panel (scale=2):**
    - gr.Markdown for detail header (request metadata)
    - gr.Row containing two gr.Columns:
      - gr.Textbox for original output, label="Original Output", lines=12, interactive=False
      - gr.Textbox for revised output, label="Revised Output", lines=12, interactive=False
    - gr.Textbox for diff highlighting, label="Changes (diff)", lines=6, interactive=False
    - gr.Textbox for reviewer name, label="Reviewer", value="operator"
    - gr.Textbox for edited response, label="Edited Response (for Edit action)", lines=5, visible=False
    - gr.Row with three buttons:
      - gr.Button("Approve", variant="primary")
      - gr.Button("Reject", variant="stop")
      - gr.Button("Edit", variant="secondary")
    - gr.Textbox for status message, label="Status", interactive=False

    c) Define callback functions:

    `def refresh_queue(rail, tenant, time_range, hide_rev):` — Calls GET /admin/hitl/queue with params. Parses response JSON. Formats each item as a row: [request_id, timestamp, tenant, triggering_rail, f"{priority:.2f}", action_taken, status_badge, prompt[:80]]. Where:
      - triggering_rail: extracted from guardrail_decisions JSON using _extract_triggering_rail logic (inline — parse all_results, find min distance)
      - action_taken: derived from guardrail_decisions (blocked/critiqued/allowed based on status_code and refusal_event)
      - status_badge: correction_action if present else "pending"
    Returns dataframe data.

    `def select_item(evt: gr.SelectData, queue_data):` — Takes selected row index, extracts request_id, calls GET /admin/hitl/queue to get the specific trace. Parses cai_critique JSON:
      - If cai_critique is not None: original_output = cai_critique["original_output"], revised_output = cai_critique["revised_output"]. Compute diff using difflib.unified_diff(original.splitlines(), revised.splitlines(), lineterm="").
      - If cai_critique is None: original_output = response, revised_output = "(Blocked before revision - no revised output available)". Diff = "".
    Returns: detail_header markdown, original_output, revised_output, diff_text, show/hide edit box.

    `def submit_correction(request_id, reviewer, action, edited_response):` — Calls POST /admin/hitl/correct with body. Returns status message.

    `def approve_click(queue_data, selected_index, reviewer):` — Gets request_id from selected row, calls submit_correction with action="approve".
    `def reject_click(queue_data, selected_index, reviewer):` — Same with action="reject".
    `def edit_click():` — Toggles edit_box visibility.
    `def submit_edit(queue_data, selected_index, reviewer, edited_response):` — Calls submit_correction with action="edit" and edited_response.

    d) Wire callbacks:
    - Refresh button click and filter changes trigger refresh_queue.
    - Queue table select event triggers select_item.
    - Approve/Reject buttons trigger their respective callbacks.
    - Edit button toggles edit_box visibility; a Submit Edit button inside edit_box triggers submit_edit.

    e) Use a gr.State component to track the currently selected request_id.

    3. Return the demo Blocks object.

    4. Handle all API errors gracefully — display error in status Textbox, do not crash. If connection refused, show "Cannot connect to harness API at {api_url}".

    5. Color-code action taken: Use the dataframe's default styling. Status badge text indicates review state ("pending", "approved", "rejected", "edited").
