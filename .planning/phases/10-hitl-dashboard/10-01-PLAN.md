    a) `async def write_correction(self, correction: dict) -> None:` — Imports redact_text from harness.pii.redactor.     c) `async def query_hitl_queue(self, since: str, rail_filter: str = "all", tenant_filter: str = "all", hide_reviewed: bool = False, limit: int = 200) -> list[dict]:` — SQL: SELECT t.*, c.action AS correction_action, c.reviewer AS correction_reviewer FROM traces t LEFT JOIN corrections c ON c.request_id = t.request_id WHERE t.timestamp >= ? AND t.guardrail_decisions IS NOT NULL AND (? = 'all' OR t.tenant = ?) ORDER BY t.timestamp DESC LIMIT ?. Parameters: (since, tenant_filter, tenant_filter, limit). Post-processing in Python:
      - Parse guardrail_decisions JSON with .get("all_results", [])
      - Compute priority using compute_priority() function (defined as module-level): For each result in all_results with score > 0, compute distance = threshold - score. Priority = 1.0 - min(distances). If no distances, priority = 0.0.
      - Extract triggering_rail as the rail name from the result with minimum distance.
      - If rail_filter != "all", skip items where triggering_rail != rail_filter.
      - If hide_reviewed and correction_action is not None, skip.
      - Parse cai_critique JSON if present.
      - Sort results: reviewed items last (correction_action is not None sorts after None), then by priority DESC.

    d) Add module-level helper: `def compute_priority(guardrail_decisions: dict) -> float:` per research Pattern 2.
    e) Add module-level helper: `def _extract_triggering_rail(guardrail_decisions: dict) -> str | None:` — returns the rail name from the all_results entry with min distance from threshold.
