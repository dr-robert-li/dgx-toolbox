    2. **GuardrailEngine class:**
       ```python
       class GuardrailEngine:
           def __init__(self, rails_config: list[RailConfig], nemo_rails=None):
               """
               Args:
                   rails_config: List of RailConfig from load_rails_config()
                   nemo_rails: Optional LLMRails instance (None in tests, real in production)
               """
               self._rails_config = {r.name: r for r in rails_config}
               self._nemo = nemo_rails
               self._input_rails = ["self_check_input", "jailbreak_detection", "sensitive_data_input", "injection_heuristic"]
               self._output_rails = ["self_check_output", "jailbreak_output", "sensitive_data_output"]
       ```

    3. **`async def check_input(self, messages, tenant, evasion_flags=None) -> GuardrailDecision`:**
       - Initialize empty `all_results: list[RailResult]`
       - For each rail name in `self._input_rails`:
         - Get RailConfig from `self._rails_config[rail_name]`
         - If `not config.enabled`: skip (do not add to all_results)
         - If rail is `"injection_heuristic"`: run `_check_injection_regex(messages)` which returns (score, detected_pattern)
         - Else: call `_run_nemo_rail(rail_name, messages)` which calls `self._nemo.generate_async()` and inspects the response for refusal text (returns score 0.0 for pass, 1.0 for block)
         - Create RailResult with rail, "pass"/"block", score, config.threshold
         - Append to all_results
       - After ALL rails run: if any result has `result == "block"`, set `blocked=True`, pick the first blocking rail as `triggering_rail`, use its `refusal_mode`
       - Build replacement_response based on refusal_mode:
         - `"hard_block"`: `_build_hard_block_refusal(triggering_rail)`
         - `"informative"`: `_build_informative_refusal(triggering_rail, reason)`
         - `"soft_steer"`: `_build_soft_steer_messages(messages)` — returns messages for second LiteLLM call (NOT the response itself — caller does the actual call)
       - Return GuardrailDecision with all fields populated

    4. **`async def check_output(self, response_data, tenant) -> GuardrailDecision`:**
       - Extract assistant content from `response_data["choices"][0]["message"]["content"]`
       - For each rail name in `self._output_rails`:
         - Same pattern as check_input: skip disabled, run rail, collect RailResult
         - For `"sensitive_data_output"`: use the existing `harness.pii.redactor.redact()` function to detect PII (if redacted text differs from original, it's a block)
       - If any block: populate `replacement_response` as a modified copy of response_data with the assistant content replaced by the refusal message (hard_block/informative) or redacted text (sensitive_data_output)
       - Return GuardrailDecision

    5. **Helper methods:**
       - `_check_injection_regex(messages) -> tuple[float, str | None]`: Join all message content, test each INJECTION_PATTERNS, return (1.0, pattern_desc) on first match, (0.0, None) on no match
       - `async _run_nemo_rail(rail_name, messages) -> float`: If `self._nemo` is None, return 0.0 (pass — no NeMo in test mode). Otherwise call `self._nemo.generate_async(messages=messages)`, inspect returned content — if it matches known refusal patterns ("I'm not able to respond", "I cannot", etc.), return 1.0 (block), else return 0.0 (pass)
       - `_build_hard_block_refusal(rail_name) -> dict`: Return `{"choices": [{"message": {"role": "assistant", "content": "I'm unable to process this request as it violates our content policy."}, "finish_reason": "stop", "index": 0}], "model": "guardrail", "usage": {"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0}}`
       - `_build_informative_refusal(rail_name, reason) -> dict`: Return response with content: `f"This request was blocked by the {rail_name} policy. Reason: {reason}. Try rephrasing your question to focus on [general guidance based on rail]."`  Map rail_name to helpful adjacent suggestions: self_check_input -> "factual, non-harmful topics", sensitive_data_input -> "questions that don't include personal information", injection_heuristic -> "direct questions without instruction override attempts", jailbreak_detection -> "questions within standard usage guidelines"
       - `_build_soft_steer_messages(original_messages) -> list[dict]`: Return `[{"role": "system", "content": SOFT_STEER_SYSTEM_PROMPT}, *original_messages]`

    6. **Module-level factory function:**
       ```python
       def create_guardrail_engine(
           rails_config_path: str,
           nemo_config_dir: str | None = None,
           litellm_base_url: str = "http://localhost:4000",
       ) -> GuardrailEngine:
           """Create a GuardrailEngine with loaded config and optional NeMo LLMRails.

           NeMo LLMRails is only created if nemo_config_dir is provided and
           nemoguardrails is importable. Otherwise engine works without NeMo
           (regex-only mode for testing and environments without NeMo).
           """
           rails_config = load_rails_config(rails_config_path)
           nemo_rails = None
           if nemo_config_dir:
               try:
                   from nemoguardrails import LLMRails, RailsConfig
                   from langchain_openai import ChatOpenAI
                   config = RailsConfig.from_path(nemo_config_dir)
                   llm = ChatOpenAI(
                       model_name="llama3.1",
                       openai_api_base=litellm_base_url,
                       openai_api_key="not-used-by-litellm",
                   )
                   nemo_rails = LLMRails(config, llm=llm)
               except ImportError:
                   pass  # NeMo not available; regex-only mode
           return GuardrailEngine(rails_config=rails_config, nemo_rails=nemo_rails)
       ```
