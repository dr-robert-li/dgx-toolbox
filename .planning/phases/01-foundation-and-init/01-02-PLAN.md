Create `modelstore/cmd/init.sh` — the interactive init wizard. Make executable (`chmod +x`).

**Header:**
```bash
#!/usr/bin/env bash
# modelstore init — Interactive setup wizard
set -euo pipefail
```

**Source libs** (relative to cmd/ location):
```bash
MODELSTORE_LIB="$(cd "$(dirname "${BASH_SOURCE[0]}")/../lib" && pwd)"
source "${MODELSTORE_LIB}/common.sh"
source "${MODELSTORE_LIB}/config.sh"
```

**Section 1: Gum detection and fallback setup (lines ~15-70)**

Check `command -v gum`. If missing, offer install from Charm apt repo (the exact install sequence from RESEARCH.md Pattern 2). Set `GUM_AVAILABLE=true/false`.

Implement three prompt helper functions:
- `prompt_input "$label" "$default" var_name` — uses `gum input --prompt "$label " --value "$default"` or `read -p "$label [$default]: "` with default fallback
- `prompt_confirm "$label"` — uses `gum confirm "$label"` (in if-then-else, not bare, to handle set -e) or `read -p "$label [y/N]: "` with `[[ "${_yn,,}" == "y" ]]`
- `prompt_choose "$label" choice1 choice2 ...` — uses `gum choose --header "$label" "$@"` or numbered menu with `read -p "Choice [1]: "`

**Section 2: Reinit detection (lines ~70-110)**

If `config_exists` is true:
1. Call `backup_config_if_exists`
2. Read old cold path: `old_cold=$(config_read '.cold_path')`
3. Print "Existing config found. Cold store: ${old_cold}"
4. Call `prompt_choose "Reinit action:"` with options:
   - "Migrate existing cold models to new cold drive"
   - "Recall everything to hot first, then configure new cold drive"
   - "Cancel"
5. Set `REINIT_ACTION` variable to "migrate", "recall_first", or exit 0
6. Show existing models with sizes before proceeding (call `show_existing_models` — defined in Section 6)

**Section 3: Hot path detection (lines ~110-140)**

Default HF hub path: `HOT_HF_DEFAULT="${HF_HOME:-${HOME}/.cache/huggingface}/hub"`
Default Ollama path: `HOT_OLLAMA_DEFAULT="${HOME}/.ollama/models"`

Display detected paths. Ask user to confirm or override each via `prompt_input`.

**Section 4: Cold drive selection (lines ~140-210)**

1. Print header: "Available drives:"
2. Run `lsblk -o NAME,SIZE,FSTYPE,MOUNTPOINT | grep -v "loop\|squash"` to show block device overview
3. If GUM_AVAILABLE: build mount list from `findmnt -o TARGET,FSTYPE,SIZE --real --noheadings`, pass to `gum choose --header "Select cold drive mount point:"`
4. If not GUM_AVAILABLE: `prompt_input "Enter cold drive mount point" "/media/${USER}" COLD_MOUNT`
5. Call `validate_cold_fs "$COLD_MOUNT"` — this will exit with error if exFAT/vfat/ntfs
6. Set `COLD_PATH="${COLD_MOUNT}/modelstore"`
7. Show `ls -la "$COLD_MOUNT"` for confirmation
8. `prompt_confirm "Use ${COLD_PATH} as cold store?"` — exit 0 if declined

**Section 5: Retention and cron config (lines ~210-240)**

1. `prompt_input "Retention period (days)" "14" RETENTION_DAYS`
2. Validate RETENTION_DAYS is a positive integer: `[[ "$RETENTION_DAYS" =~ ^[0-9]+$ ]] || ms_die "Retention must be a number"`
3. `prompt_input "Cron hour (0-23, daily migration)" "2" CRON_HOUR`
4. Validate CRON_HOUR is 0-23: `[[ "$CRON_HOUR" =~ ^[0-9]+$ ]] && (( CRON_HOUR >= 0 && CRON_HOUR <= 23 ))` or die
5. `prompt_input "Config backup retention (days)" "30" BACKUP_RETENTION_DAYS`

**Section 6: Model scan display (lines ~240-340)**

Implement `scan_hf_models()`:
- Walk `${HOT_HF_PATH}/models--*/` directories
- For each: extract model name (strip `models--` prefix, replace `--` with `/`), get size via `du -sb`, get last-modified via `stat --format="%Y"`
- Format with `printf "%-50s %10s %12s\n"` for MODEL, SIZE, LAST USED columns
- Convert bytes to human-readable with `numfmt --to=iec-i --suffix=B` (fallback: awk)
- Convert epoch to date with `date -d "@${epoch}" "+%Y-%m-%d"`
- Track and print subtotal

Implement `scan_ollama_models()`:
- Walk `${HOT_OLLAMA_PATH}/manifests/registry.ollama.ai/library/` for model manifests
- For each manifest file: extract model:tag from path, sum blob sizes via `jq -r '.layers[].digest | gsub(":"; "-")' "$manifest"` then `stat --format="%s"` each blob
- Same printf table format
- Track and print subtotal

Implement `show_existing_models()`:
- Print header "Current models on hot storage:"
- Call scan_hf_models and scan_ollama_models
- Print grand total

Call `show_existing_models` after drive paths are confirmed.

**Section 7: Confirmation and directory creation (lines ~340-390)**

1. Print summary of all settings in a formatted block
2. `prompt_confirm "Create directory structure and save config?"` — exit 0 if declined
3. `mkdir -p "${HOME}/.modelstore/usage"` — state dir first (before writing config!)
4. Create cold drive directories:
   ```bash
   mkdir -p "${COLD_PATH}/huggingface/hub"
   mkdir -p "${COLD_PATH}/ollama/models"
   ```
5. Call `write_config "$HOT_HF_PATH" "$HOT_OLLAMA_PATH" "$COLD_PATH" "$RETENTION_DAYS" "$CRON_HOUR" "$BACKUP_RETENTION_DAYS"`

**Section 8: Crontab installation (lines ~390-420)**

Implement `install_cron()`:
- Build cron entries: `0 ${CRON_HOUR} * * * ${MODELSTORE_DIR}/cron/migrate_cron.sh` and `30 ${CRON_HOUR} * * * ${MODELSTORE_DIR}/cron/disk_check_cron.sh`
- Where MODELSTORE_DIR resolves to the absolute modelstore/ directory
- Remove old modelstore entries: `crontab -l 2>/dev/null | grep -v "modelstore" || true`
- Append new entries via pipe to `crontab -`
- Print confirmation message

Call `install_cron "$CRON_HOUR"`.

**Section 9: Reinit actions (lines ~420-460)**

If `REINIT_ACTION` is set:
- "migrate": `rsync -av --info=progress2 "${old_cold}/" "${COLD_PATH}/"` then prompt to cleanup old dir
- "recall_first": Print message that recall is a Phase 3 feature, suggest running `modelstore recall --all` after Phase 3

Print "Init complete. Config saved to ~/.modelstore/config.json"
